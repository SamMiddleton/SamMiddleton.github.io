Problem Solving

Getting Blocked

I once had to convert some code that I had written with NumPy to use PyTorch. It involved finding the maximum value of an array, and I suddenly got garbage output.

Through a combination of debug prints, trying things out with the interpereter and reading the documentation, I learned that NumPy and PyTorch have different naming conventions for the various ways you can interperet "finding the maximum". Overall, I first felt confused, gradually changing to annoyed.

An Elegant Solution

I've done a bunch of work on making a custom Keypoint Detection neural network in PyTorch based on the provided KeypointRCNN. This required creating my own Dataset to load the images and annotations. This took a little bit of reading the documentation to work out what format I needed to present the data in, but it was fairly easy to inferit from TorchVision's provided datasets. I also created another Dataset that formatted the output differently for an attempt to create a model not based on KeypointRCNN.

I then wanted to create versions of those Datasets that pulled the images from s3. This took a bit more reading of documentation to work out how to do, as well as some planning on how I wanted to cache the images so I didn't download them multiple times. I then realised that I was using the same code in both the modified Datasets, and thought I could use multiple inheritance to avoid repeating myself. After a bit more research, I was able to pull it off and felt extremely smug. I learnt that I was capable of using multiple inheritance to keep my code DRY.

CSS Positioning Modes

If you don't have a scrollbar, you should resixe this window until you do.

The CSS position property has seven possible values:

  1. static
  2. fixed
  3. relative
  4. sticky
  5. absolute
  6. initial
  7. inherit

The default value is static, meaning that the element's position is determined by where it appears in the HTML.

Using fixed makes an element stay in the same place in the window, even when you scroll. The top, bottom, left & right properties are used to determine how far away from each edge the element is placed. The element does not leave a gap where it would normally be in fixed mode.

Using absolute position makes top, bottom, left & right position the element relative to the bounding box of the closest ancestor with a non-static position, or otherwise the body of the page (as is the case withthe list above). The element will scroll with the page (unless inside something that doesn't) and won't leave a gap where it would normally be.

Using relative allows you to nudge an element around. In this mode, setting left: 20px; will insert 20 pixels of space to the left of the element. Other elements will be positioned as if this element occupied its default position.

An element with sticky position behaves as if it was relative unless it scrolls to the edge of the window, where it will behave as if it was fixed. To make sticky work in Safari, you need to use -webkit-sticky instead.

Using initial sets any property to its default value, in this case static static. Similarly, inherit sets any property to the value the parent has, also static in this case. These values were mainly included in the list to show how relative & sticky leave space.
